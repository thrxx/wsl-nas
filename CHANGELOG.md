## [1.0.0] — 2026-04-21
### Добавлено
- Первый релиз автоматизированного набора для WSL2 Self-Hosted NAS
- Скрипты автоматической настройки: `01_setup_wsl.sh` → `05_setup_backup.sh`
- Оптимизированный `smb.conf` с принудительным SMB2.10+, отключённым гостевым доступом и фильтрацией системного мусора (`veto files`)
- Шаблон `wsl.conf` с `systemd=true` и монтированием с поддержкой метаданных (`umask=22,fmask=11`)
- Автоматизация сети: `Update-WSLPortProxy.ps1` для динамического проброса портов + правила Брандмауэра Windows (только профиль `Private`)
- Автоматизация бэкапов: инкрементальная синхронизация `rsync` + шифрованное версионирование `restic` + сжатие VHDX (`wsl --manage --compact`)
- Диагностика состояния: `04_health_check.sh` с проверкой сервисов, портов, конфигов и места на диске
- Полная документация: Архитектура (4 уровня I/O), Сеть, Безопасность, Стратегии бэкапа, Устранение неполадок
- CI через GitHub Actions: ShellCheck (Bash) + PSScriptAnalyzer (PowerShell)
- Шаблоны задач: Отчёт об ошибке и Запрос функции со встроенными чек-листами диагностики

### Исправлено
- Исправлен синтаксис heredoc в исходных черновиках (`cat > >` / `< <` → `cat << 'EOF'`)
- Исправлены опечатки в PowerShell-командлетах (`Joi n-Path`, `A ction`, `De lay`)
- Устранён конфликт `set -e` с арифметикой в Bash (`((PASS++))` → `PASS=$((PASS + 1))`)
- Добавлена безопасная обработка `crontab` для идемпотентного планирования бэкапов
- Стандартизированы переменные путей (удалены лишние пробелы)

### Безопасность
- Принудительно установлен `server min protocol = SMB2_10` (SMBv1 окончательно отключён)
- Запрещён анонимный/гостевой доступ по умолчанию (`map to guest = never`, `guest ok = no`, `restrict anonymous = 2`)
- Правило Брандмауэра ограничено профилем `Private` (защита от публичных сетей)
- Задокументировано явное разделение контекстов аутентификации (Windows / Linux / `smbpasswd`)
- Добавлены предупреждения против публикации порта 445 в интернет; рекомендованы альтернативы: Tailscale/VPN
- Отключён `interop` в `wsl.conf` для предотвращения случайного запуска Windows-бинарников из Linux-оболочки

### Примечания
- Решение предназначено **только для домашнего/лабораторного использования**. Не заменяет TrueNAS/OMV/Unraid.
- WSL2 засыпает вместе с хостом Windows; отсутствует поддержка аппаратного RAID и ИБП.
- Динамический диск VHDX автоматически расширяется, но требует периодического сжатия.
- Активные данные NAS **должны** храниться в нативном `/data` (ext4), чтобы избежать потерь производительности протокола 9P.
- Все скрипты идемпотентны: безопасно запускать повторно без риска повредить конфигурацию.
