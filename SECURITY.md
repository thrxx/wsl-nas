## 🏗️ Встроенные меры безопасности (Security by Default)

В этом проекте по умолчанию применяются следующие настройки для защиты ваших данных:

### 1. Сетевая изоляция
*   **SMBv1 отключен:** В конфигурации Samba принудительно установлен `server min protocol = SMB2_10`. Это защищает от атак WannaCry/EternalBlue.
*   **Гостевой доступ запрещен:** Параметры `map to guest = never` и `guest ok = no` включены по умолчанию. Анонимный вход в систему невозможен.
*   **Брандмауэр Windows:** Скрипт `03_configure_windows.ps1` создает правило фаервола **только для профиля `Private`**. Открытие портов в публичных сетях заблокировано.

### 2. Аутентификация
*   **Разделение контекстов:** Проект использует отдельную базу паролей Samba (`smbpasswd`), которая не связана с системными паролями Linux или Windows. Это предотвращает эскалацию привилегий при взломе одной из учетных записей.
*   **Валидация конфигов:** Перед запуском сервисов скрипт запускает `testparm -s`, чтобы убедиться, что в `smb.conf` нет синтаксических ошибок, открывающих бреши.

## ⚠️ Обязанности пользователя (Важно!)

Поскольку это Self-Hosted решение, безопасность ваших данных критически зависит от настройки вашей среды:

### 🚫 Критически запрещено
1.  **Проброс порта 445 в Интернет:** Никогда не пробрасывайте порт 445 напрямую из роутера в интернет (DMZ/Port Forwarding). SMB не предназначен для публичных сетей. Для удаленного доступа используйте VPN (Tailscale, WireGuard).
2.  **Использование профиля сети `Public`:** Убедитесь, что ваша сеть в Windows определена как "Частная" (Private). В публичной сети Windows блокирует большинство SMB-соединений, и скрипты могут работать некорректно.
3.  **Хранение данных на `/mnt/c`:** Файлы в `/mnt/c` используют протокол 9P и права NTFS. Это может привести к утеке данных или повреждению прав доступа. Всегда используйте нативный том `/data` (ext4).

### ✅ Рекомендуемые действия
1.  **Шифрование диска:** Поскольку виртуальный диск WSL (`ext4.vhdx`) хранится в `AppData\Local\Packages...`, настоятельно рекомендуется включить **BitLocker** на системном диске Windows. Это защитит файлы NAS в случае кражи ноутбука/диска.
2.  **Сложные пароли:** Используйте уникальные пароли для пользователя Samba, отличающиеся от пароля учетной записи Windows.
3.  **Отключение Interop:** Если вам не нужно запускать `.exe` файлы из Linux-терминала, настройте `/etc/wsl.conf` (`interop.enabled = false`), чтобы предотвратить запуск вредоносного ПО Windows из окружения NAS.
