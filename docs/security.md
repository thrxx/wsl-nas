# 🔒 Безопасность и Hardening WSL-NAS

> ⚠️ **Область применения:** Решение оптимизировано для изолированной домашней или лабораторной сети. Оно не сертифицировано для enterprise-сред, не поддерживает аппаратное шифрование дисков и не заменяет специализированные системы контроля доступа. Критически важные данные должны храниться только в сочетании с регулярными шифрованными бэкапами.

## 🔐 1. Аутентификация: три независимых контекста
WSL-NAS использует разделённую модель учётных записей. Понимание различий предотвращает 90% ошибок доступа.

| Контекст | Где хранится | Для чего используется |
|----------|--------------|------------------------|
| **Windows User** | SAM / Active Directory | Вход в ОС, доступ к `\\wsl$` (медленно, через 9P) |
| **Linux User (WSL)** | `/etc/passwd`, `/etc/shadow` | Права на файлы в ext4, выполнение скриптов, `sudo` |
| **Samba User** | `/var/lib/samba/private/passdb.tdb` | **Только** для сетевого подключения по SMB/CIFS |

⛔ **Критическое правило:** Пароли **не синхронизируются** автоматически. 
```bash
# Правильная настройка доступа по сети:
sudo smbpasswd -a $USER  # Задаёт пароль ТОЛЬКО для Samba
```
💡 Рекомендуется использовать уникальный пароль для сетевого доступа, отличный от пароля Windows/Linux.

## 🌐 2. Сетевая безопасность и изоляция
SMB/CIFS исторически не предназначен для работы в ненадёжных сетях. Открытие порта `445` вне локального сегмента создаёт критическую уязвимость.

| Практика | Статус | Обоснование |
|----------|--------|-------------|
| `server min protocol = SMB2_10` | ✅ Включено | Отключает уязвимый SMBv1 (EternalBlue, WannaCry) |
| Брандмауэр Windows: `Profile Private` | ✅ Обязательно | Блокирует входящие SMB из гостевых/публичных сетей |
| Проброс 445 на внешний IP роутера | ❌ Запрещено | SMB не шифрует трафик, раскрывает структуру ФС, подвержен MITM |
| Удалённый доступ из интернета | ✅ Только VPN/Tailscale | WireGuard/Tailscale шифрует трафик и скрывает порты |

### 🔒 Рекомендуемая архитектура удалённого доступа
```
[Телефон/Ноутбук вне дома] → Tailscale / ZeroTier (WireGuard)
                                 ↓
                        [Ваш ПК в LAN: 100.x.x.x]
                                 ↓
                        \\100.x.x.x\data (SMB поверх зашифрованного туннеля)
```

## 🛡️ 3. Hardening Samba & WSL
Параметры, минимизирующие поверхность атаки и предотвращающие эскалацию прав.

| Параметр / Действие | Значение | Эффект |
|---------------------|----------|--------|
| `map to guest = never` | В `smb.conf` | Запрещает анонимный вход. Требует валидных учётных данных |
| `guest ok = no` | В шар `[data]` | Отключает гостевой доступ к папкам |
| `restrict anonymous = 2` | В `smb.conf` | Запрещает перечисление пользователей и общие ресурсы без аутентификации |
| `force user = %S` | В `smb.conf` | Файлы создаются от имени подключившегося пользователя, а не `root` |
| `interop.enabled = false` | В `/etc/wsl.conf` | Блокирует вызов `.exe` из Linux, снижает риски выполнения неавторизованных host-процессов |
| Регулярные обновления | `sudo apt update && wsl --update` | Закрывает CVE в ядре Linux, Samba и компонентах WSL |

## 📁 4. Защита данных и целостность
Безопасность NAS определяется не только сетевой защитой, но и устойчивостью к повреждению/шифрованию данных.

| Угроза | Мера защиты |
|--------|-------------|
| **Ransomware / Удаление** | `restic`/`borg` с версионированием и WORM-настройками. См. [backup.md](backup.md) |
| **Повреждение метаданных** | Хранение данных **только** в `/data` (ext4). Избегать `/mnt/c/` (протокол 9P не гарантирует атомарность прав) |
| **Сбой питания / Зависание Windows** | `sync always = no` + `strict sync = no` (производительность), но обязательные инкрементальные бэкапы каждые 24ч |
| **Утечка через `\\wsl$`** | Отключить общий доступ к папке в Проводнике. Использовать только SMB-шару для клиентов |

## 🚨 5. Модель угроз и ответные меры
| Вектор атаки | Вероятность | Влияние | Митигация в WSL-NAS |
|--------------|-------------|---------|---------------------|
| Сканирование портов в LAN | Средняя | Низкое | SMB требует аутентификации, `restrict anonymous = 2` скрывает ресурсы |
| Подбор пароля Samba | Низкая | Высокое | `smbpasswd` использует отдельную БД, рекомендуется сложность ≥12 символов |
| Эксплуатация SMB-уязвимостей | Низкая | Критическое | SMBv1 отключён, `server min protocol = SMB2_10`, регулярные `apt upgrade` |
| Повреждение `.vhdx` при сбое | Средняя | Высокое | Динамический VHDX устойчив к journaling ext4. Бэкапы + `wsl --manage --compact` |
| Утечка данных через публичный Wi-Fi | Высокая (при ошибке настройки) | Критическое | Брандмауэр `Private` по умолчанию, запрет проброса 445 в WAN |

## ✅ Чек-лист безопасности (перед вводом в эксплуатацию)
- [ ] `guest ok = no` и `map to guest = never` активны в `smb.conf`
- [ ] `server min protocol = SMB2_10` (проверено через `testparm -s | grep "server min protocol"`)
- [ ] Брандмауэр Windows разрешает TCP 445 **только** для профиля `Private`
- [ ] Пароли Samba заданы через `smbpasswd -a`, не совпадают с системными
- [ ] В `/etc/wsl.conf` отключено `interop` (опционально, но рекомендовано)
- [ ] Данные хранятся в нативном `/data`, а не в `/mnt/c/` или `\\wsl$`
- [ ] Настроен минимум 1 инкрементальный бэкап с проверкой восстановления
- [ ] Нет правил NAT/Port Forwarding на роутере, направляющих 445 в интернет
- [ ] WSL и пакеты обновлены (`wsl --update`, `sudo apt upgrade`)
